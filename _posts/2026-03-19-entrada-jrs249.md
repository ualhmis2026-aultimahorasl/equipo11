# Despliegue del Blog en Azure Static Web Apps

**Fecha:** 19 de marzo de 2026  
**Autor:** Jorge Ramírez Sánchez

---

## Introducción

Una vez que tenemos nuestro blog desarrollado y funcionando en local, el siguiente paso es desplegarlo para que cualquiera pueda accederlo desde internet. En esta entrada explico cómo lo hicimos usando **Azure Static Web Apps**, el servicio en la nube de Microsoft para alojar sitios web estáticos de forma gratuita.

---

## ¿Qué es Azure Static Web Apps?

Azure Static Web Apps es un servicio de Microsoft Azure que permite alojar sitios web estáticos (HTML, CSS, JavaScript) de forma sencilla y gratuita en su capa básica. Se integra directamente con GitHub, lo que significa que cada vez que hacemos `push` a la rama principal, el sitio se actualiza automáticamente.

---

## Paso 1: Crear la Web App en Azure

1. Accede al portal de Azure en [https://portal.azure.com](https://portal.azure.com) con tu cuenta de estudiante.
2. Haz clic en **"Crear un recurso"** y busca **"Aplicación web"**.
3. Rellena los datos de configuración:
   - **Suscripción:** Azure para Estudiantes
   - **Grupo de recursos:** crea uno nuevo, por ejemplo `hmis-tuusuario`
   - **Nombre:** `hmis-tuusuario` (debe ser único globalmente)
   - **Runtime stack:** Node 18 LTS (o la versión más reciente disponible)
   - **Sistema operativo:** Linux
   - **Región:** Central US
   - **Plan de App Service:** Free (F1) — importante para no gastar créditos
4. Haz clic en **Revisar y crear** y espera unos segundos. Tu URL estará disponible en `https://hmis-tuusuario.azurewebsites.net`.

---

## Paso 2: Configurar el despliegue con Git local

En el portal de Azure, dentro de tu Web App:

1. Ve a **Centro de implementación** en el menú lateral.
2. En la pestaña **Configuración**, selecciona **Git local** como origen.
3. Guarda los cambios. Azure generará una **URI de Git Clone** y unas credenciales automáticas.
4. Copia la URI y las credenciales desde la pestaña **Credenciales de Git o FTPS**.

> ⚠️ El nombre de usuario es solo `$hmis-tuusuario`, sin el prefijo `hmis-tuusuario\` que a veces aparece delante.

---

## Paso 3: Cambiar la rama de despliegue a `main`

Por defecto Azure usa la rama `master`. Como nuestro repositorio usa `main`, hay que cambiarlo:

1. Ve a **Configuración** → **Nueva configuración de la aplicación**.
2. Añade:
   - **Nombre:** `deployment_branch`
   - **Valor:** `main`
3. Guarda los cambios.

---

## Paso 4: Desplegar desde el repositorio local

En la terminal, dentro de la carpeta de tu repositorio local, añade Azure como un segundo remoto de Git:

```bash
git remote add azure URL_DE_GIT_CLONE
```

Luego haz push a ese remoto:

```bash
git push azure main
```

Se te pedirán las credenciales que copiaste en el paso anterior. Si todo va bien, en unos segundos el blog estará disponible en tu URL de Azure.

---

## Resultado

Nuestro blog quedó desplegado y accesible públicamente. Cada vez que el responsable fusiona un pull request en `main`, basta con hacer `git push azure main` para actualizar el sitio, sin necesidad de entrar al portal de Azure.

---

## Conclusión

Azure Static Web Apps es una opción muy cómoda para proyectos académicos: es gratuita, se integra bien con Git y no requiere gestionar ningún servidor. La curva de aprendizaje inicial es baja y el resultado es un despliegue profesional en la nube.
